Collecting Sensor Data from Smart Devices

Project Description | FAQ

Project Description

The object of this project is to gather sensor data of various type from smart devices. We plan to analyze the collected sensor data to characterize different aspects of user and device behavior from a security and privacy point of view.

To gather sensor data from smart devices we have developed an android app that basically collects sensor data and periodically uploads the data to a remote server. The source code for the app if available here.

We are careful in not violating user privacy and are therefore not tracking any personal data. Our project went through IRB review and since we are not collecting any personal data of users IRB office has confirmed that we do not require any IRB approval. The official IRB letter is available here .

This experiment is in collaboration with several researchers, but the leads are Anupam Das, a Ph.D. student at the University of Illinois at Urbana-Champaign, and Zahid Rahman, a Ph.D. student at the Indiana University at Bloomington. This project is being supervised by Professor Nikita Borisov (Hatswitch Group) and Professor Matthew Caesar from the University of Illinois at Urbana-Champaign.

Feel free to contact us with any question or suggestion.

What do the buttons on the app do?
The App has the following three buttons-
  1. Start: starts the sensor reading process.
  2. Stop: stops the sensor reading process but the application doesn't exit.
  3. Exit: Terminates the app.
Once the app is started you can press the "Home" button to run the app in the background. However, if you press the "Back" button the process will stop (has the same effect as clicking the "Stop" button inside the app). At any point you can stop and restart the app by clicking the "Stop" and "Start" button respectively.
How do I run the app?
We would like participants to the run the app under the following scenarios:
  1. While the device is stable on some plain surface. For example, assume you are sitting on a chair with your smartphone lying on the table.
  2. While the device is in some periodic motion. For example your walking/jogging with your smartphone in your pocket.
  3. While the device is in some irregular motion. For example you are talking with your friend in speaker-mood where the smartphone may be moved in an irregular pattern.
How long will this take?
We would like to gather data for at least 5-10 minutes for each of scenarios described above. In other words, we would like to obtain enough data to characterize each of the above described scenarios separately.
How much bandwidth, external disk space, RAM, and CPU will this app consume?
The size of the app is 263KB. Once installed it consumes 1.01MB of disk space.

We tested our app on the following two sets-

  1. LG Nexus 4.
  2. Galaxy Note 2.

We found the following average CPU and RAM requirements-

Device Properties CPU RAM
LG Nexus 4 (Quad core, 1500 MHz, Krait) 10-25 % 30 MB
Samsung Galaxy Note 2 (1.6GHz Quad-Core Processor) 10-20 % 15 MB

The app generates one log file for each sensor where each log file is around 1MB in size. So at any given point in time if the device has WiFi connection then the app will use only few MB (~10MB) of space on the SD card. If WiFi connection is not available the data files are stored on the SD card in which case the space required is proportional to the amount of time it runs. We would encourage the user to run the app when it is connected to some WiFi network, so that the storage requirement is minimal.

On our tests with the two sets we found our app uploading 8MB of data every 4 minutes, so the average bandwidth required is ~40KB/s.

What kind of permissions are required to run the app?
The App requires the following permissions-
  1. Storage : Temporarily stores the sensor readings in SD card before uploading them to a remote server. Once uploaded the files are deleted.
  2. Network Communication: Needed to upload data to remote server. And check if WiFi connection is available.
  3. Phone Identity: Needed to uniquely identify data from different devices. Identity information is hashed before sending to the remote server. So exact identity is not used. If you have any privacy concern please look inside the source code.
Where is the data stored?
Data are temporarily saved inside the SD card, but are deleted once the file is uploaded. For each sensor the app creates a logging file and whenever it finishes logging 25000 records it uploads the file to the remote server. Each file is about ~1MB in size. Sensor readings are sent out to the following server- datarepo.cs.illinois.edu. We plan to make the data public to other researchers in future.

*We would like to point out that the app uploads data only when there is WiFi connection, otherwise it storage the files in SD card. So you may see a lot of files that are temporarily stored in absence of WiFi connection. But once it detects WiFi connection it starts uploading all previously stored files..

What is being measured and why?
We are measuring the sensor values under different settings. We hope to the analyze the sensor data to uniquely characterize different aspects of user and device behavior. Such characterization would be useful from both security (are you using the right device) and privacy (what information are smart devices leaking) point of view.
